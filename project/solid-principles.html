• SOLID belongs to 5 different types of principles we are using in project architecture
  1. S: Single Responsibility Principle (SRP)
    • Single Responsibility Principle says that "Every software module should have only one reason to change".
    • This means that every class, in our code should have only one job to do.
    • Everything in that class should be related to a single purpose.
    • For Eg: If we have 2 requirements like to register the user and send email to the user.
      In this case we will have 2 seperate services called UserService and EmailService.
      Not everything in a single file
  2. O: Open/Closed Principle
    • The Open/closed Principle means "A class is open for extension and closed for modification".
    • "Open for extension" means, we need to design our classes, like a new functionality can be added only when new requirements are needed. (In C# we can use inheritance to do this)
    • "Closed for modification" means we already developed a class and we have written unit testing.
      We should not modify those classes until we find any bugs. (Default Repository, can accept any type of objects).
